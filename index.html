ESP32 Sterilization Monitor

Medical-grade embedded system demonstrating real-time control, safety interlocks, and deterministic state machine design.

Project Concept

I wanted to build a sterilization cycle controller that demonstrates production-style embedded system design principles used in medical devices. The goal was to create a dual-MCU architecture with proper separation between safety-critical control logic and user interface components—mirroring the fault isolation strategies found in certified medical equipment.

The system would implement FreeRTOS task management, deterministic state machine control, real environmental sensing, hardware safety interlocks, and UART-based telemetry—all fundamental concepts in safety-critical embedded applications.

Components

All components for the dual-MCU sterilization monitor

I started by gathering the components needed for both the controller unit and the user interface subsystem. The controller would handle all safety-critical operations, while a separate ESP32 display module would provide real-time visualization without impacting control performance.

Controller ESP32-S3-WROOM-1 for state machine and sensor management
UI Module ESP32 2.8" display board for real-time visualization
Sensors BME280 I²C temperature and humidity sensor
Safety Normally-closed door switch, relay module, status LEDs, piezo buzzer
Control Momentary push button for cycle initiation

Building the Prototype

Initial breadboard prototype with ESP32-S3 controller

I assembled the first prototype on a breadboard, connecting the ESP32-S3 to the BME280 sensor over I²C, wiring up the relay module for heater control simulation, and integrating the safety interlock switch. The status LEDs and buzzer were connected to provide immediate visual and audible feedback based on the system state.

The architecture uses a strict separation of concerns—the controller ESP32-S3 runs all safety-critical tasks including the state machine, sensor reading, interlock monitoring, and output control. Meanwhile, the display ESP32 receives structured telemetry over UART and focuses solely on visualization.

Testing & Integration

With the hardware assembled, I began developing the firmware using ESP-IDF and FreeRTOS. The controller firmware implements five main tasks with appropriate priority levels: cycle control for the state machine, sensor task for I²C communication, safety handler for interlock monitoring, output control for relay and indicators, and UART telemetry for UI updates.

The display module required its own firmware to receive and parse the UART telemetry stream. I developed a simple protocol to transmit state, temperature, humidity, and timing information at regular intervals. The display shows the current cycle state, live sensor readings, and elapsed time—providing a clean interface without impacting the controller's real-time performance.

State Machine Implementation

Door interlock emergency stop demonstration during active cycle

The core of the controller is a deterministic finite state machine with explicit transitions and comprehensive fault handling. Each state has clear entry and exit conditions, ensuring predictable and auditable behavior throughout the sterilization cycle.

The FAULT state can be entered from any operational state when safety violations are detected—such as door opening during a cycle or sensor communication failures. This defensive approach ensures the system fails safely and provides clear feedback about what went wrong.

Final Result

System demonstration: complete sterilization cycle from IDLE to COMPLETE state

The completed system successfully demonstrates production-grade embedded design principles applicable to medical devices and industrial control systems. It features real-time task scheduling with FreeRTOS, deterministic state machine execution, environmental sensor integration with validation, multi-MCU communication over UART, and hardware-enforced safety interlocks with immediate shutdown capability.

While the heating profile is simulated in software for safety and repeatability, the system validates live sensor data and provides a realistic foundation for further development. Future enhancements could include MQTT logging, SD card audit trails, OTA firmware updates, and integration with a Linux-based companion system for advanced analytics.
