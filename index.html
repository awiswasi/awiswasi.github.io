Project Concept

This project was designed to demonstrate production-style embedded system architecture commonly found in medical devices. A dual-MCU approach isolates safety-critical control logic from user interface responsibilities.

The system integrates FreeRTOS task scheduling, deterministic state-machine control, real environmental sensing, hardware interlocks, and UART telemetry.

Components

Controller and UI subsystem components

An ESP32-S3-WROOM-1 serves as the primary controller, responsible for executing the state machine, managing sensors, and enforcing safety logic. A separate ESP32-based display module acts purely as a UI endpoint, receiving structured telemetry over UART.

Environmental feedback is provided by a BME280 sensor over I²C. Safety mechanisms include a normally-closed door interlock, relay-based heater simulation, status LEDs, an audible buzzer, and a momentary push button for cycle control.

Building the Prototype

Initial breadboard prototype

The first prototype was assembled on a breadboard, integrating the ESP32-S3 with the BME280 sensor, relay module, safety interlock, and indicator hardware.

Safety-critical tasks run exclusively on the controller MCU, while the display MCU handles visualization without affecting real-time behavior.

Testing & Integration

Firmware was developed using ESP-IDF and FreeRTOS, with dedicated tasks for cycle control, sensor acquisition, safety monitoring, output control, and UART telemetry.

The display module parses telemetry packets and renders state, temperature, humidity, and elapsed time independently of the controller.

State Machine Implementation

Emergency stop via door interlock

The controller is driven by a deterministic finite state machine with explicit transitions and defensive fault handling. The cycle progresses through IDLE, PRECHECK, HEAT-UP, STERILIZE, COOL-DOWN, and COMPLETE states.

A FAULT state can be entered from any operational state if safety violations occur, ensuring the system fails safely and provides clear diagnostic feedback.

Final Result

Complete sterilization cycle demonstration

The completed system demonstrates production-grade embedded design principles, including deterministic control flow, real-time scheduling, and hardware-enforced safety interlocks.

While the heating profile is simulated, the architecture provides a strong foundation for future expansion and real-world integration.
