ESP32 Sterilization Monitor

Medical-grade embedded system demonstrating real-time control, safety interlocks, and deterministic state machine design.

Project Concept

I wanted to build a sterilization cycle controller that demonstrates production-style embedded system design principles used in medical devices. The goal was to create a dual-MCU architecture with clear separation between safety-critical control logic and user interface components.

The system implements FreeRTOS task management, deterministic state-machine control, real environmental sensing, hardware safety interlocks, and UART-based telemetry—core concepts in safety-critical embedded applications.

Components

All components for the dual-MCU system

The controller is built around an ESP32-S3-WROOM-1, responsible for running the state machine, handling sensor input, and enforcing all safety logic. A separate ESP32-based 2.8" display module acts purely as a user interface, receiving structured telemetry over UART.

Environmental data is provided by a BME280 temperature and humidity sensor over I²C. Safety is handled through a normally-closed door interlock switch, a relay module for heater control simulation, status LEDs, and an audible piezo buzzer. A single momentary push button is used to initiate and control the sterilization cycle.

Building the Prototype

Initial breadboard prototype

The first prototype was assembled on a breadboard, connecting the ESP32-S3 to the BME280 sensor via I²C, wiring the relay module for heater simulation, and integrating the safety interlock switch.

The controller ESP32-S3 runs all safety-critical tasks, while the display ESP32 focuses exclusively on visualization—ensuring that UI updates never interfere with real-time control behavior.

Testing & Integration

Firmware was developed using ESP-IDF and FreeRTOS. The controller runs multiple tasks with defined priorities, including cycle control, sensor acquisition, safety monitoring, output control, and UART telemetry.

The display firmware parses the telemetry stream and renders the current cycle state, live sensor data, and elapsed time without impacting controller performance.

State Machine Implementation

Emergency stop via door interlock

The controller is driven by a deterministic finite state machine with explicit transitions and defensive fault handling. The cycle progresses through IDLE, PRECHECK, HEAT-UP, STERILIZE, COOL-DOWN, and COMPLETE states.

A FAULT state can be entered from any operational state if safety violations occur, ensuring the system fails safely and provides clear diagnostic feedback.

Final Result

Complete sterilization cycle demonstration

The completed system demonstrates production-grade embedded design principles applicable to medical and industrial control systems, including real-time task scheduling, deterministic control logic, and hardware-enforced safety interlocks.

The heating profile is simulated for safety, but the system processes live sensor data and provides a solid foundation for future expansion.
